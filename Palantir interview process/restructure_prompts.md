## PROMPT 10: Write New Chapter 19 — Technical Comprehension

Write a new file at:
`/Users/clayskaggs/Developer/general textbooks/Palantir Decomposition Interview/chapter_19_technical_comprehension.md`

This chapter prepares for the **Technical Comprehension** interview round. From the recruiter:
- Interviewer introduces a technical concept you've NEVER seen before
- You solve problems of increasing complexity using that concept
- You are NOT expected to know the material in advance
- It's collaborative — ask lots of questions
- SQL basics recommended as prep

### Chapter Structure:

**19.1 What This Round Is** — Format: interviewer teaches → you apply → problems get harder. Simulates real Palantir work. Multiple valid approaches exist.

**19.2 The Learning Ladder** — How to build understanding live:
1. Listen without interrupting
2. Restate in your own words
3. Ask boundary/edge-case questions
4. Build a mental model (sketch, analogies)
5. Solve the simple case first
6. Extend incrementally

**19.3 How to Ask Great Questions** — "Can you give me a concrete example?", "Is this similar to [X]?", "What's the expected output for [edge case]?", "Am I on the right track?"

**19.4 When and How to Pivot** — If stuck >2 min, say so. Pivoting shows flexibility.

**19.5 SQL — The Most Likely Technical Domain** — Review basics (SELECT, WHERE, JOIN, GROUP BY, HAVING, ORDER BY, aggregates). Cover likely "new concept" candidates: window functions, CTEs, recursive queries, self-joins. Plain English first, technical term in parentheses.

**19.6 Practice Exercises** — 3 exercises: "You've never seen [concept]. Here's how it works: [explanation]. Now solve this problem using it: [problem]." Topics: window functions, CTEs, one non-SQL concept.

**19.7 Chapter Summary — Quick Reference Card**

Use same tone as rest of textbook: plain English, technical terms in parentheses, concrete examples, no fluff. Aim for ~600-800 lines.
